# Six-Class Slit-Lamp Image Classification Benchmark

This repository contains the preprocessing, augmentation, training, evaluation,
and visualization code for a six-class slit-lamp image classification
benchmark.

## Data and Weight Availability

The clinical images are not publicly released because they contain sensitive
patient-derived information and are subject to institutional privacy and ethics
restrictions. Study-trained classification and YOLO11 segmentation weights are
also not publicly distributed.

To support methodological reproducibility, this repository provides the
complete code pipeline, dependency versions, model and preprocessing
configurations, augmentation settings, and fixed de-identified split metadata.
Third-party pretrained weights must be obtained from their official sources and
used according to their licenses.

## Repository Contents

```text
.
├── README.md
├── requirements.txt
├── preprocess_conjunctiva.py
├── preprocessing_config.json
├── data_augmentation.py
├── train.py
├── checkpoint_manager.py
├── evaluate_bootstrap.py
├── model_configs.json
└── deidentified_split_metadata.csv
```

`preprocess_conjunctiva.py` is the preprocessing implementation used in the
YOLO11 instance-segmentation pipeline. `model_configs.json` records model
architectures, input sizes, normalization, online augmentation, optimization
settings, and adaptation protocols. `deidentified_split_metadata.csv` contains
anonymous patient/image IDs, class labels, development/test cohort membership,
fixed validation-fold assignments, and original/augmented image type; it
contains no original identifiers or filenames.

## Installation

Reference environment: Python 3.10, PyTorch 2.8.0, torchvision 0.23.0, and
CUDA 12.8.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Expected Local Layout

Authorized clinical data and legally obtained third-party pretrained weights
should be arranged locally as:

```text
data/
├── train/
├── train_transformed/
└── test/
weights/
├── dinov2/model.safetensors
├── dinov3/
├── ctranspath/ctranspath.pth
└── clip-vit-base-patch32/
outputs/
```

Image directories contain the class subdirectories `NC`, `BK`, `FK`, `VK`,
`NIFK`, and `AMK`. Files containing `_aug_` are treated as augmented training
images.

## Preprocessing and Augmentation

YOLO11 conjunctiva segmentation weights must be supplied through `--weights`.
Images without a predicted mask are reported and skipped. Exact mask
post-processing and segmentation training reference settings are recorded in
`preprocessing_config.json`.

```bash
python preprocess_conjunctiva.py \
  --input-dir data/raw \
  --output-dir data/train \
  --weights /path/to/conjunctiva_yolo11_best.pt

python data_augmentation.py \
  --input-dir data/train \
  --output-dir data/train_transformed \
  --seed 42
```

## Training

Training uses fixed-seed grouped five-fold cross-validation. Validation folds
contain original images only; development-fold training uses original and
offline-augmented images. Cross-entropy loss with label smoothing is used
without class weighting, balanced sampling, or focal loss. All benchmarked
trainable models share the same optimization baseline: AdamW, learning rate
`3e-4`, weight decay `1e-4`, and `ReduceLROnPlateau`. This unified baseline
applies to the CNN backbones, ViT, DINOv2, DINOv3, CTransPath, and CLIP
prompt-based models whenever they are trained (`linear_probe` or
`full_finetune`). The only exception is CLIP `zero_shot`, which does not train
model parameters and therefore skips the optimizer and scheduler. Model-
specific input preprocessing and augmentation remain unchanged.

```bash
python train.py \
  --config model_configs.json \
  --train-dir data/train_transformed \
  --test-dir data/test \
  --output-dir outputs/checkpoints \
  --models ResNet18 \
  --protocol full_finetune
```

Distributed foundation-model training example:

```bash
torchrun --nproc_per_node=4 train.py \
  --config model_configs.json \
  --train-dir data/train_transformed \
  --test-dir data/test \
  --output-dir outputs/checkpoints \
  --models DINOV2 \
  --protocol full_finetune
```

## Evaluation and Visualization

Evaluation requires checkpoints generated by `train.py`; study-trained
checkpoints are not distributed.

```bash
python evaluate_bootstrap.py \
  --base-ckpt-dir outputs/checkpoints \
  --val-root data/test \
  --output-dir outputs/evaluation \
  --eval-mode fivefold_test \
  --models ResNet18 \
  --bootstrap-iters 1000 \
  --seed 42
```

Run any script with `--help` to view its configurable paths and options.
